# Design: Positive Photo Dispenser (긍정 사진 배달부)

Generated by /office-hours on 2026-04-16
Branch: main
Status: DRAFT
Mode: Builder

## Problem Statement
사용자가 입력하는 텍스트에서 부정적인 감정을 감지하여, 즉각적으로 긍정적인 사진을 보여줌으로써 기분 전환과 웃음을 선사합니다.

## What Makes This Cool
- **즉각성:** 고민을 적는 순간 바로 웃음이 터지게 만드는 빠른 피드백.
- **직접 준비한 사진:** 사용자의 취향이 반영된 큐레이션된 긍정 에너지.

## Constraints
- **기술 스택:** Vanilla HTML/JS/CSS (외부 의존성 없음).
- **감지 방식:** 키워드 매칭 (미리 정의된 부정적 단어 목록).

## Premises
1. 사용자가 텍스트를 입력하는 동안 실시간으로 단어를 검사합니다.
2. 특정 키워드가 발견되면 즉시 `./photos/` 폴더 내에서 미리 준비된 사진들 중 하나를 골라 화면에 띄웁니다.
3. 사진은 빌더가 직접 제공하며, 파일 경로를 배열에 담아 프로그램이 참조할 수 있게 합니다.
4. 복잡한 설정 없이 브라우저에서 파일(index.html)을 여는 것만으로 동작해야 합니다.

## Approaches Considered
### Approach A: 초간단 실습형 (Vanilla JS + 키워드 매칭) - **SELECTED**
- 최소한의 코드로 핵심 기능 구현.
- 빌더가 직접 제공하는 사진 파일들을 `./photos/` 폴더에 넣고 경로를 설정하여 사용.

### Approach B: 지능형 웹 서비스 (React + Gemini API)
- 문맥 분석을 통한 정교한 감정 파악 (차후 업그레이드 대상).

## Recommended Approach
**Approach A**를 통해 가장 빠른 프로토타입을 구축합니다. HTML의 `<textarea>`와 `keyup` 이벤트를 활용해 실시간성을 확보하고, CSS Transition으로 시각적 완성도를 높입니다.

## Open Questions

- 얼마나 많은 키워드를 초기 목록에 넣을 것인가?

## Success Criteria
- 부정적인 단어 입력 시 0.5초 이내에 사진이 화면에 나타남.
- 사진을 본 사용자가 웃음을 지을 수 있는 따뜻한 레이아웃.

## Next Steps
1. `index.html` 파일 생성 및 기본 레이아웃 구성.
2. 부정적 키워드 사전(`keywords.js`) 작성.
3. 사진 노출 로직 구현 및 CSS 효과 추가.

## What I noticed about how you think
- **"사진만 띄워도 돼"**: 본질에 집중하는 능력이 탁월하십니다. 복잡한 기능보다 사용자에게 전달할 '가치(웃음)'를 우선시하는 것이 훌륭한 빌더의 자세입니다.
- **"사진은 내가 제공할게"**: 자신의 콘텐츠에 대한 애착과 준비성이 좋습니다. 이는 프로젝트가 단순한 기술 연습을 넘어 '자기 것'이 되게 만드는 핵심 요소입니다.
